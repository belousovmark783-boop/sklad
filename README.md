# Система управления складом товаров

Веб-приложение для управления складским помещением, включая отслеживание размещения товаров на стеллажах, управление контрактами клиентов и мониторинг сроков хранения.

## Возможности

- 📦 **Управление товарами** — добавление, удаление и отслеживание товаров
- 🏢 **Управление помещениями** — создание и управление складскими помещениями
- 📋 **Управление стеллажами** — организация стеллажей в помещениях с расчётом вместимости
- 👥 **Управление клиентами** — ведение информации о клиентах и их банковских реквизитах
- 📊 **Статистика** — общая информация о количестве товаров, помещений, стеллажей и клиентов
- ⏰ **Мониторинг сроков** — отслеживание истекших контрактов на хранение
- 🔌 **REST API** — JSON API для получения данных о складе

## Технологический стек

- **Backend:** Django 3.2+
- **Database:** PostgreSQL (psycopg2)
- **Environment:** python-dotenv

## Модели данных

### Room (Помещение)
- `name` — Название помещения
- `capacity` — Полезный объём (м³)

### Rack (Стеллаж)
- `number` — Номер стеллажа
- `room` — Ссылка на помещение
- `slots_count` — Количество мест
- `slot_height` — Высота места (м)
- `slot_width` — Ширина места (м)
- `slot_length` — Длина места (м)
- `max_load` — Максимальная суммарная нагрузка (кг)

### Client (Клиент)
- `name` — Наименование юридического лица
- `bank_details` — Банковские реквизиты

### Item (Товар)
- `height`, `width`, `length` — Размеры товара (м)
- `weight` — Вес товара (кг)
- `arrival_date` — Дата поступления
- `contract_number` — Номер договора
- `client` — Ссылка на клиента
- `contract_end_date` — Дата окончания договора
- `rack` — Ссылка на стеллаж
- `position` — Позиция размещения (номер)

## Установка

### Требования
- Python 3.8+
- PostgreSQL 10+

### Шаги установки

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd sklad
```

2. Создайте виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. Создайте файл `.env` в корне проекта:
```
SECRET_KEY=your-secret-key-here
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1
DATABASE_URL=postgresql://user:password@localhost:5432/warehouse
```

5. Выполните миграции:
```bash
cd ozon
python manage.py migrate
```

6. Создайте суперпользователя:
```bash
python manage.py createsuperuser
```

7. Запустите сервер разработки:
```bash
python manage.py runserver
```

Приложение будет доступно по адресу `http://localhost:8000`

## API Endpoints

### GET `/api/data/`
Возвращает JSON со всеми данными о складе:
- `stats` — Статистика (количество помещений, стеллажей, клиентов, товаров)
- `rooms` — Список помещений со стеллажами
- `racks` — Список всех стеллажей
- `clients` — Список клиентов с инициалами
- `items` — Список товаров с информацией о сроках и состоянии

**Пример ответа:**
```json
{
  "stats": {
    "rooms": 5,
    "racks": 20,
    "clients": 15,
    "items": 150
  },
  "rooms": [...],
  "racks": [...],
  "clients": [...],
  "items": [...]
}
```

### GET `/`
Главная страница с отображением всех данных о складе
